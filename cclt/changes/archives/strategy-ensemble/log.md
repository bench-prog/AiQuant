## 技术决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 组合方式 | 信号加权融合 | Regime 路由 / 置信度叠加 | 最灵活，权重可随时间调整 |
| 子策略 | AI + Trend | + Regime | 互补性最强，实现复杂度适中 |
| 权重 | 固定 60/40 | Hyperopt | 简单起步，后续可扩展 |
| 信号归一化 | ai_prediction 原生 [0,1] + trend 映射 [0,1] | 其他映射 | 统一可比较 |
| 出场逻辑 | ensemble_score < threshold | 任一子策略触发 | 与入场对称，逻辑清晰 |

## 知识发现

- [x] **策略组合模式**: 在单一 IStrategy 中集成多个子策略的信号逻辑，通过加权融合生成最终信号 → `strategy_ensemble_v1.py`
- [x] **信号归一化**: 不同策略的信号范围不同（概率 [0,1] vs 布尔 {0,1}），必须统一映射才能加权 → `strategy_ensemble_v1.py::_compute_trend_signal`
- [x] **互补性设计**: AI 抓时机 + Trend 抓方向，两者在不同市场状态下各有优势，组合后平滑收益曲线 → `strategy_ensemble_v1.py::ENSEMBLE_WEIGHTS`
