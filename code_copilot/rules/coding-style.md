# 编码规范

## 1. 命名

- 函数名：小写 + 下划线（`snake_case`），动词开头
- 类名：大驼峰（`PascalCase`）
- 常量：全大写 + 下划线分隔
- 模块名：小写 + 下划线
- 私有函数/变量：单下划线前缀
- 禁止拼音、中英混拼命名

## 2. 类型注解

- 所有函数参数和返回值必须加类型注解
- 复杂类型使用 `from typing import ...`
- DataFrame 参数标注 `pd.DataFrame`

## 3. 异常处理

- 业务异常使用自定义异常类
- 禁止吞掉异常（空 `except:`）
- 异常中必须记录日志
- 涉及外部 API（ccxt/交易所）必须有超时和重试

## 4. 日志

- 使用 Python `logging` 模块
- 策略入口打 INFO
- 异常打 ERROR，含完整堆栈
- 禁止在日志中打印 API Key、密钥

## 5. pandas / numpy 规范

- 优先向量化操作，禁止逐行 `df.iterrows()` 处理大数据
- 缺失值处理必须显式（`fillna()` / `dropna()`），不可隐式传播
- 时间序列操作注意索引对齐，防止数据泄漏
- 特征计算后必须验证无 `inf` / `nan`

## 6. 其他

- 写策略必须考虑回测和实盘的一致性
- 涉及并发场景必须说明同步策略
- 魔法值必须定义为常量
- 每个模块顶部写简短 docstring 说明用途
