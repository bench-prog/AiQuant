# 安全红线

## 1. 代码安全

- ❌ 禁止在代码中硬编码 API Key、Secret、数据库密码
- ❌ 禁止提交包含个人隐私的测试数据
- ❌ 禁止在日志中打印 API Key、Secret

## 2. 业务安全

- ⚠️ 涉及交易逻辑（下单、撤单、仓位调整）必须在 spec 中明确标注，人工审查后方可编码
- ⚠️ 涉及策略参数/风控阈值变更，必须检查回测一致性
- ⚠️ 涉及模型文件加载，必须验证来源可信

## 3. 量化交易专项安全

- ⚠️ `dry_run: true` 是默认状态，切换为实盘必须经过代码审查
- ⚠️ 止损/风控逻辑不可被跳过或注释掉
- ⚠️ 模型推理结果必须经过 sanity check（如概率范围、NaN 检测）
- ⚠️ 资金费率/持仓成本计算必须与实际交易所规则一致
